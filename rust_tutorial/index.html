What is Rust?

Rust（公式サイト）は、Mozillaが開発する 高速性、安定性、生産性を主軸とした静的なプログラミング言語です。

言語としては比較的新しく（2006年誕生、2015年にバージョン1.0リリース）、現在（2019年）でも活発に開発がおこなわれています。 バージョンは６週間に一度更新され、多くの機能が積極的に取り入れられているのも特徴です。

主な機能

他の言語にはあまりない、Rustならではの機能には以下のようなものがあります。

  • 所有権システム
    • 所有権
    • 参照と借用
    • ライフタイム
  • 強力な型システム
    • トレイト
    • ジェネリクス

なじみのない概念も多いかと思いますが、ゆっくりと理解していきましょう。

Why Rust?

　RustはC言語・C++といった低レイヤー・システム開発用の言語の置き換えをねらった言語ですが、 Webサーバーなどのバックエンド、WebAssemblyを用いたアプリケーションなどのフロントエンドの記述にも適しています。

　大企業での採用実績もあります。Dropboxでは自社製のストレージシステムMagic PocketをRustで開発しています （ The Epic Story of Dropbox's Exodus From the Amazon Cloud Empire）。もちろん、開発元のMozillaも Firefox QuantumでパーサーをRustで書き直すことで、パフォーマンスの改善を行っています。

　積極的にRustに乗り換える理由には以下のものがあります：

安全性

　C言語やC++には制約が少なく、ヒープにメモリを確保する、ポインタでメモリにアクセスする、 ポインタの値を変更する、などといった危険なことも可能でした。その反面、 危険なことをするときには、開発者はそのすべてにおいて責任を負う必要があり、 それを怠るとよく困ったことが起こるのでした。たとえば、

  • ダングリングポインタ
  • スタックの破壊
  • メモリリーク
    • など...

です。これらは、開発者の手違いや不注意によって生まれ、一度生じてしまうと、 多くの時間がその解決に割かれてしまうのでした。

　Rustでは、そういった危険な操作をunsafeの砦に追いやることで、普段使いのコーディングをより安全に行うことができます。

パフォーマンス

　Rustは静的な言語で、コンパイル時にほとんどの挙動を確定します。これは、実行時にはじめてパースされるJavaScriptや Pythonに比べて大いに有利で、コンパイルは行うが動的なランタイムを有するC#やGoと比べても高速です。 ベンチマーク結果 （ixy-languages/ixy-languages） では、C言語に比類するパフォーマンスを出すこともあります。

　Rustはゼロコスト抽象化を掲げています。要するに、抽象化の実現に 実行時にコストが掛かるものは極力排そう、という考え方です。 たとえば、ポリモーフィズムは Rustではトレイト、Java、C#、Goなどではインターフェイスによって実現されます。 インターフェイスは、vtableやリフレクションなどによって実装されているため、 呼び出しなどの際にコストがかかります。
　これに対して、Rustのトレイトでは、静的に型を確定できる場合にはコンパイル時に解決するので、実行時にコストが かかりません。 （ちなみに、トレイトの動的な呼び出しを可能にするトレイトオブジェクトという言語機能もありますが、 奨励はされていません。）

　また、コンパイラバックエンドにLLVMを採用し、多くのプラットフォームに対して高速なプログラムを生成することが出来ます。

生産性

　Rustには、cargoとよばれるパッケージ管理・ビルドシステムが付属しています。 テストやベンチマークの機能もあり、ビルドスクリプトによる高度なビルドも行えます。

　また、Rustにはクレートとよばれるライブラリ群があり、自前で実装したり 環境構築したりすることなく、Cargo.tomlに加筆するだけですぐに必要な機能を追加することが出来ます。 クレートの多くはMIT、またはMIT + Apache 2.0のデュアルライセンス（Rust自体とおなじライセンス）なので、 GPLによるライセンス汚染の可能性はとても低く、商用にも安心して利用できます。

　当サークル（MIS.W）の一部では、「公用語をRustにしよう」という動きもあります。[要出典]

インストール

インストールはとても簡単です。

rustup経由のインストール

rustup（rustup.rs）経由でのインストールが一般的です。

Windows以外の環境では、 

$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

を実行するだけです。Windowsの場合は、rustup.rsの指示に従えばいいでしょう。

さらに、いくつかのコンポーネントを追加でインストールしておくといいでしょう。 

$ rustup component add clippy
$ rustup component add rustfmt
$ rustup component add rls
アップデート

Rustのアップデートは、rustup経由で行うことが出来ます。 

$ rustup update
COLUMN: リリースチャネル

Rustにはstable、beta、nightlyの3つのチャネルが存在します。 （How Rust is Made and “Nightly Rust”）

リリースチャネル リリース頻度 特徴
stable 6週間に一度 もっとも安定したバージョン。通常つかうのはこちら。
beta 6週間に一度 stableリリース前のバージョン。安定化に向けてnightlyから分かれて、バグ修正がおこなわれる。
nightly 毎日 開発中のバージョンで、masterブランチにあたる。新しい機能を使うときはこれをつかう。

Hello, world!

Rustのインストールがおわったところで、実際に動くかどうか試してみましょう。cargoで 新しいプロジェクトを作成してみましょう。 

 $ cargo new hello-world
    Created binary (application) `hello-world` package

cargo newコマンドによって、現在の作業ディレクトリにhello-worldというディレクトリが 作成されました。中を見てみると、"src/main.rs"に 

fn main() {
    println!("Hello, world!");
}


と、書かれていますね。では、これを実行してみましょう。プログラムの実行にはcargo runコマンドを使用します。 

 $ cargo run
    Compiling hello-world v0.1.0 (/../../../hello-world)
     Finished dev [unoptimized + debuginfo] target(s) in 3.99s
      Running `target/debug/hello-world`
Hello, world!

無事、Hello, world!が出ました。環境構築お疲れさまでした。

型と構文

Rustには様々な型や、構文があります。一通り見ていきましょう。

さまざまな型

Rustには大きく分けて、

  • プリミティブ型
  • 参照・ポインタ
  • 構造体
  • 列挙型

があります。

プリミティブ型

最も基本的な型です。

名前 型 例
整数型 u32, i64, usize... 1123, 0xCAFEBABE, -273, 0
浮動小数点型 f32, f64 3.14, -1.414, 6.02e23
論理値型 bool true, false
配列 [T; n] [2, 7, 3, 8]
スライス &[T] &a[1..4]
char型 char 'a' '🐱'
str型 str "nyan"
タプル (i32, &str)... (2019, "世話やきキツネの仙狐さん")
関数型 fn(f64) -> f64... fn id(x: f64) -> f64 { x }
参照・ポインタ

参照やポインタを用いることで、値そのものを移動せずに扱うことが出来ます。mutable（変更可能）な参照（ポインタ）と immutable（変更不能）な参照（ポインタ）があります。詳しくは後のほうで解説します。 

let x = 30;
let y = &mut x;
*y = 40;

構造体

いくつかのフィールドを持つ、または一切のフィールドをもたない型です。

pub struct Point2D {
    pub x: f64,
    pub y: f64,
}

struct Senko(String);

struct Serval;
列挙型

複数の状態を持つ型です。それぞれの状態がフィールドをもつこともできます。

enum NPCState {
    Idle,
    Chase(Rc<RefCell<Target> >),
    Attack { target  : Rc<RefCell<Target> >,
             strategy: Box<AttackStrategy> },
}
さまざまな構文

言語単位

Rustの言語単位は、

  • 文（statement）
    • 変数束縛
    • コードブロック
    • 式（expression）
      • 制御式（if、for、while、loop）
      • 直定数（literal）
      • 中置記法演算（infix operation）
        • など...
    • 関数宣言
    • 型定義
という感じになっています。

関数定義

プログラムは、一連の手続きを記したものです。手続きは、関数の中に記します。

関数定義は、fn 関数名(引数名: 型, ...) -> 返り値の型 {ブロック} の形で行います。C言語などと違い、型は後置です。 

fn sum(a: i64, b: i64) -> i64 {
    return a + b;
}

セミコロンのない式は、返り値として解釈されます。

fn sum(a: i64, b: i64) -> i64 {
    a + b
}
関数呼び出し

関数名に、括弧で囲まれた引数が続きます。

sum(33, 4); // 37
## 変数束縛 `let`を使うことで、値を名前に束縛することが出来ます。 ```rust let x: i32 = 1123; ``` こんな感じ。` let 変数名: 型（ = 値） ` の形。 型が分かりきっているものについては、コンパイラが勝手に予測してくれます。 ```rust let phi = 0.1257; // <- f64 ``` でも、再代入は出来ません。 ```rust let x: i32 = 1123; x = 2; ``` ```console $ cargo build ... error[E0384]: cannot assign twice to immutable variable `x` --> src/main.rs:3:5 | 2 | let x: i32 = 1123; | - first assignment to `x` 3 | x = 4; | ^^^^^ cannot assign twice to immutable variable error: aborting due to previous error For more information about this error, try `rustc --explain E0384`. error: Could not compile `helloworld`. To learn more, run the command again with --verbose. ``` これは、Rustでの変数が標準でimmutable（不変値）となっているからです。じゃあ、どうやったら変更可能になるかというと、 ```rust let mut x: i32 = 0; x = 420; // ok ``` というふうに、`mut`（mutable: 変更可能）をつければ変更可能になります。 ### ムーブセマンティクス ```rust let a: String = "けものフレンズ".to_string(); let b: String; b = a; println!("{}", a); println!("{}", b); ``` `b`へ`a`が代入される時、`b`は初期化されていないので代入に成功します。しかし、コンパイルが通りません。なぜでしょう。 ```console $ cargo build error[E0382]: use of moved value: `a` --> src/main.rs:7:20 | 5 | b = a; | - value moved here 6 | 7 | println!("{}", a); | ^ value used here after move | = note: move occurs because `a` has type `std::string::String`, which does not implement the `Copy` trait error: aborting due to previous error For more information about this error, try `rustc --explain E0382`. error: Could not compile `helloworld`. To learn more, run the command again with --verbose. ``` Rustでは、一つの値は一つの変数と対応します。そしてコピーできる一部の型(※)を除き、値を代入した時点で所有権が代入先に移されます。詳しくはまたいつかどこかで。 ※`Copy`トレイトが実装されているものはコピーされます。ムーブされるものを複製したいときには、大抵`Clone`トレイトを実装していることが多いので`**.clone()`してあげましょう。 ## if文 Rustのif文（式）には条件式の括弧がありません。 ```rust if is_lazy() { println!("しんどい"); } else { println!("しんどくない"); } ``` `let`と組み合わせて、パターンマッチングすることも出来ます。 ```rust let a: Option = Some(114514); if let Some(val) = a { println!("{}", val); // 114514 } ``` 式としても使えます。 ```rust let username = if let Some(n) = get_name() { n } else { "名無し" }; ``` ※ちなみにこの場合は、`unwrap_or`を使う方が楽です。 ```rust let username = get_name().unwrap_or("名無し"); ``` ## match文 ざっくり説明すると、高機能なswitch文。 ```rust let coffee: u32 = 2; match coffee { 0 => println!("カフェインが足りないですぅ..。"), 1 => println!("ねむっちゃいそう...。"), 2 => println!("お仕事頑張りますっ！"), _ => println!("（カフェイン中毒）（心不全）（死）"), } ``` パターンマッチもできます。 ```rust let mut state: NPCState = NPCState::Idle; ... match &state { NPCState::Idle => { ... }, ... NPCState::Attack { target: t, strategy: s } => { s.try_attack(t); }, } ``` ## ループ ### for文 範囲指定は`a..b`（a < i < b）や`a..=b`（a < i ≦ b）のように指定します。 ```rust let a = ["キタキツネ", "ギンギツネ", "フェネック"]; for i in 0..a.len() { println!("{}はフレンズなんだね！", a[i]); } ``` イテレータを使うこともできるよ。 ```rust let a = ["マーゲイ", "オセロット", "ライオン", "ジャガー"]; for f in &a { println!("{}はフレンズなんだね！", f); } ``` ### while文 言うまでも無くwhile文です。 ```rust while is_lazy_cat() { println!("しんどい"); } ``` ### loop文 条件式を含まない単純なループもあります。 ```rust loop { println!("大学やだーーーーー！！！！"); } ```
